## possible classes

## game loop
  - while either player 1 or 2 score is not zero
  - prompt current player for input (answer to question)
  - if answer is wrong, decrease score
  - if score is zero, display game over (including player scores)

## variables to track
  - current player
  - player scores
  - user input
